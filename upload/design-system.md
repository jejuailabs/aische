# design-system.md — 비주얼 디자인 & 커스터마이징

## 1. 디자인 톤

기본은 **깔끔한 프로덕티비티 툴 UI**를 베이스로 하되, 사용자가 원하면 게임화(캐릭터 아바타 + 배경 테마)를 통해 화려한 몰입형 경험으로 커스터마이징할 수 있는 **하이브리드 구조**.

- 기본 모드: 미니멀 카드형 UI (다크/라이트 모드 대응 용이)
- 커스텀 모드: 배경 이미지 위에 반투명 카드 오버레이 (사용자 제공 목업의 화려한 톤)

## 2. 커스터마이징 대상

### 2.1 아바타 (AI 비서 캐릭터)
- 기본 제공: 사전 생성된 샘플 아바타 갤러리 (AI 이미지 생성 API로 미리 만들어 Firebase Storage에 저장 — **실시간 온디맨드 생성 아님, 운영비 관리를 위해 사전 생성 방식**)
- 사용자 확장: 직접 이미지 업로드 가능
- 표현 방식: 1차 버전은 **정적 이미지 + 원형 프레임** (음성 반응 시 파형/원형 애니메이션은 프레임 외곽 이펙트로 별도 처리, 아바타 자체는 애니메이션 없음)
- 향후 확장: Lottie 또는 Rive 기반 애니메이션 아바타 (표정/립싱크 등) — 스펙에는 포함하되 구현은 후순위(Phase 6 이후)

### 2.2 배경 테마
- 기본 제공: 사전 생성 샘플 배경 세트 (Storage 저장)
- 사용자 확장: 직접 이미지 업로드
- 다크/라이트 모드와의 공존: 배경 이미지가 설정된 경우, 다크/라이트 토글은 **카드 UI의 불투명도·텍스트 대비값만 조정하는 오버레이 방식**으로 처리 (배경 이미지 자체를 다크/라이트로 바꾸지 않음)

### 2.3 저장 원칙
아바타/배경 등 모든 이미지 에셋은 **Firebase Storage에 저장**하고 `users/{uid}/preferences.avatarAssetRef`, `backgroundAssetRef`로 참조. 로컬 파일이나 빌드 번들에 포함하지 않는다 (`core.md` §2-2 원칙 재확인).

## 3. 음성 인터페이스 비주얼

- 중앙 원형 아바타 프레임
- 마이크 활성화 시 원형 외곽에 오디오 레벨 반응 파형/원 확장 애니메이션 (`ai-voice-input.md` §3)
- 이 애니메이션 레이어는 아바타 이미지와 분리된 별도 SVG/Canvas 레이어로 구현 → 아바타 교체와 무관하게 항상 작동

## 4. 게임화 요소 (확장 여지)

- 포인트/코인 시스템 (사용자 목업의 우상단 포인트 표시 참고)
- 프로젝트 진행률에 따른 배지/레벨업 요소
- 이 항목들은 Phase 6 이후 별도 스펙으로 구체화

## 5. 다크 / 라이트 모드

- 전역 토글 스위치 (헤더 또는 설정)
- 디자인 토큰 기반 (CSS 변수)으로 관리하여 모든 컴포넌트에 일괄 적용
- 사용자 설정은 `UserPreferences.theme`에 저장, 기기 시스템 설정과 별개로 앱 내 설정이 우선

## 6. 카테고리 색상 (캘린더 연동)

`data-model.md`의 `Category.color`와 연결되는 색상 팔레트는 라이트/다크 모드에서 각각 대비가 확보되도록 이중 값(라이트용/다크용 HEX)으로 관리한다.

## 7. 반응형 원칙

- PC: `ui-layout.md`의 3단 대시보드
- 모바일: 하단 탭바 기반 단일 컬럼
- 브레이크포인트 기준 컴포넌트 재배치 (그리드 위젯 → 세로 스택 전환 등)
