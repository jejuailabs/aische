# ui-layout.md — PC 대시보드 & 모바일 내비게이션 구조

## 1. PC 대시보드

### 1.1 레이아웃 구조 (3단)

```
┌─────────┬──────────────────────────────┐
│ 도메인   │                                │
│ 메뉴     │      워크스페이스 영역          │
│ (전역)   │  (모듈을 유저가 추가/제거/배치)  │
│─────────│                                │
│ 프로젝트 │   [투두 모듈] [캘린더 모듈]     │
│ 메뉴     │   [만다라트 모듈] ...          │
│ (트리)   │                                │
└─────────┴──────────────────────────────┘
```

- **좌측 상단 — 도메인 메뉴 (전역 내비게이션)**: 캘린더 / 투두 / 만다라트 / 설정 / 관리자 등 앱 전체 섹션 이동
- **좌측 하단 — 프로젝트 메뉴**: 만다라트 프로젝트 트리 목록. 클릭 시 워크스페이스에 해당 프로젝트 컨텍스트가 반영됨
- **중앙 — 워크스페이스**: 위젯형 모듈(투두/캘린더/만다라트/간트 등)을 사용자가 드래그로 추가·제거·배치. `react-grid-layout` 등으로 구현하며, 배치 상태는 `data-model.md`의 `UserPreferences.pcWorkspaceLayout`에 저장

### 1.2 참고 레퍼런스 (사용자 제공 목업 반영)

사용자가 제시한 여행앱 목업(제주 CCTV 앱)의 구조를 레이아웃 레퍼런스로 채택:

- 좌상단 "오늘의 일정" 미니 타임라인 → 캘린더 위젯과 동일 개념
- 중앙 원형 아바타 + 마이크 → 음성 인터페이스 위젯 (`ai-voice-input.md`)
- 우측 "오늘의 투두" 패널 → 투두 위젯
- 우측 "프로젝트 트리(만다라트)" 계층 진행률 패널 → 만다라트 위젯
- 하단 탭 형태 카테고리 → 워크스페이스 프리셋/필터 개념으로 응용

단, 이 목업의 **테마 종속 콘텐츠(여행 CCTV, 추천 장소 등)와 풀스크린 배경 이미지 위 카드 방식은 그대로 채택하지 않고**, 구조(위젯 배치 + 중앙 AI + 우측 트리 패널)만 채택한다. 화려한 비주얼 톤은 `design-system.md`의 커스텀 배경/아바타 기능으로 사용자가 선택적으로 구현할 수 있게 한다.

### 1.3 위젯 종류 (초기 셋)

- 캘린더 위젯 (월간 미니뷰)
- 투두 위젯 (오늘 할 일)
- 만다라트/프로젝트 트리 위젯 (진행률 요약)
- AI 비서 위젯 (음성/텍스트 대화창)
- 알림/로그 위젯 (Phase 6)

## 2. 모바일 내비게이션

### 2.1 구조 — 하단 탭바 + 스와이프 병행

- **하단 탭바**: 캘린더 / 투두 / 만다라트 / 대시보드(또는 더보기) 아이콘으로 명시적 이동
- **스와이프**: 각 화면에서 좌우 스와이프 시 탭바 순서대로 인접 화면 전환 (탭바 순서 = 스와이프 순서, 별도의 상하좌우 고정 매핑 없음)
- 화면 안에서 세부 View 전환(예: 리스트 ↔ 간트)이 필요한 경우 상하 스와이프 대신 **화면 내 뷰 전환 버튼**을 사용 (제스처 충돌 방지 및 직관성 확보)

### 2.2 홈 모드 설정

앱을 열었을 때 기본으로 보이는 첫 화면(홈 탭)을 사용자가 지정:

- 캘린더 우선
- 대시보드 우선
- 만다라트 우선

`data-model.md`의 `UserPreferences.homeMode`에 저장.

### 2.3 음성 인터페이스 접근

모바일에서는 하단 탭바 중앙 또는 플로팅 버튼으로 마이크 아이콘을 상시 노출하여, 어느 화면에 있든 즉시 음성 입력 가능 (`ai-voice-input.md` §3 UX와 동일).
