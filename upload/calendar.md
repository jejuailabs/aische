# calendar.md — 캘린더 모듈

## 1. 개요

구글 캘린더 느낌의 기본 틀을 따르되, 두 가지 뷰를 핵심으로 한다.

1. **월간 뷰 (기본 진입 화면)**
2. **일간 상세 뷰 (특정 날짜 탭 시 확장)**

데이터 소스는 `data-model.md`의 `Node`(`type: "calendar_event"`) 컬렉션이며, 캘린더는 이 데이터의 하나의 View일 뿐이다.

## 2. 월간 뷰

- 한 달(예: 6월) 전체 날짜 칸(보통 7×5~6)이 한 화면에 동시에 표시
- **각 날짜 칸 내부에 자체적으로 06:00~22:00 / 30분 단위의 미니 세로 타임라인이 들어감**
  - 칸이 작기 때문에 30분 단위 눈금선을 물리적으로 다 그리지 않는다.
  - 실제 시간 계산은 30분 단위로 정확히 하되, **화면에는 눈금선 없이 해당 시간대 비율 위치에 색상 구간(bar)만 표시**한다.
  - 목적은 정밀한 판독이 아니라 "대략 몇 시쯤 무슨 색 일정이 있다"는 정도의 인지다.
- 색상은 일정의 `category`(카테고리)에 따라 결정되며, 캘린더 상단 여백에 **색상-텍스트 범례(legend)**로 표시
- 특정 날짜 칸을 탭/클릭하면 해당 날짜의 **일간 상세 뷰**로 확장 전환

## 3. 일간 상세 뷰

- 06:00~22:00, 30분 단위 눈금선이 선명하게 표시되는 세로 타임라인
- 각 일정 블록의 상세 정보(제목, 시간, 장소, 대상자 등) 노출
- 일정 블록 탭 시 상세 편집/삭제/완료 처리 가능

## 4. 카테고리 범례

- `data-model.md`의 `Category` 컬렉션 기반
- 사용자가 카테고리 추가/색상 변경 가능
- 범례는 월간 뷰 상단 빈 공간에 칩(chip) 형태로 노출, 탭하면 해당 카테고리만 필터링해서 보기 가능

## 5. 필수 입력 필드 (일정 등록 시)

일정 생성 시 아래 필드가 기본으로 요구되며, AI 입력 경로(텍스트/음성/파일 등)를 통해 들어올 때도 동일 기준 적용 (`ai-voice-input.md` 참조):

- 날짜
- 시간 (몇 시부터 몇 시간)
- 대상자
- 장소
- 내용

부족한 필드가 있으면 AI가 사용자에게 되물어 채운 뒤, 컨펌을 받아야 정식 등록된다 (Draft → Confirm).

## 6. 알림

- 일정당 최대 5개까지 알림 등록 가능
- 단위: 시간 / 일 / 주 / 월 (`data-model.md`의 `Reminder[]`)
- 예: "1일 전 + 1시간 전 + 정시" 조합 등록 가능

## 7. 투두/만다라트 연동

- 일정은 투두 항목과 자연스럽게 연동(`todo.md` 참조)
- 일정은 AI 분석을 통해 기존 만다라트 프로젝트에 자동 배치 제안되며, 승인 후 반영 (`mandarat.md` §16)
- 매칭되지 않는 일정은 미분류함에 모여 사용자가 직접 정리

## 8. 다중 View 동기화

캘린더에서 수정한 내용은 만다라트/투두/간트/리스트 View에도 즉시 반영된다 (동일 Node 데이터를 참조하므로 별도 동기화 로직 불필요, Firestore 실시간 리스너로 처리).
