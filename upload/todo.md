# todo.md — 투두 모듈

## 1. 벤치마킹 요약 (Todoist / TickTick 기준, 2026년 기준 조사)

시장 최고 수준 투두 앱 비교 결과, 두 축으로 기능이 갈린다.

- **Todoist형(심플·AI 강점)**: <cite index="2-1">깔끔한 인터페이스와 강력한 자연어 입력, 목표를 실행 단계로 쪼개주는 AI 기능, 음성을 구조화된 태스크로 변환하는 기능, 이메일을 태스크로 자동 전환하는 기능, 자연어로 필터를 생성하는 기능</cite>을 갖춤. <cite index="6-1">특히 날짜·반복 일정을 파싱하는 자연어 입력 성능이 시장에서 가장 뛰어난 편</cite>으로 평가됨.
- **TickTick형(올인원 기능 밀도)**: <cite index="2-1">뽀모도로 타이머, 습관 트래커, 아이젠하워 매트릭스, 통합 캘린더 뷰를 기본 번들로 제공</cite>하며, <cite index="6-1">드래그로 태스크를 옮기고 캘린더 일정과 나란히 보여주는 타임라인 뷰</cite>가 강점.
- <cite index="4-1">시각적 긴급도 표시, 카운트다운, 뽀모도로, 태스크를 의도적으로 '하지 않음' 처리해 부담을 줄이는 기능, 커스터마이징 가능한 뷰·테마·색상</cite> 등이 사용자 만족도가 높은 요소로 꼽힘.

→ 우리 시스템은 **Todoist의 자연어/AI 입력 강점 + TickTick의 올인원 뷰(캘린더 통합) 강점**을 모두 흡수하는 방향으로 설계한다. 이미 `core.md`에서 정의한 "단일 데이터 모델 + Draft/Confirm + 다중 View" 구조가 이 두 장점을 자연스럽게 포괄한다.

## 2. 데이터 모델

`data-model.md`의 `Node`(`type: "todo"`) 그대로 사용. 투두 고유 필드는 없고, 캘린더/만다라트와 동일한 스키마를 공유한다 — 이것이 벤치마킹 대상 앱들과의 핵심 차별점(투두=캘린더=목표가 하나의 데이터).

## 3. 핵심 기능 목록

### 3.1 캡처(입력)
- 텍스트 자연어 입력 (날짜/반복/우선순위 자동 파싱)
- 음성 입력 (`ai-voice-input.md` 참조)
- 이메일/카톡 공유 → 태스크 자동 전환
- 빠른 추가(Quick Add) — 상세 화면 이동 없이 리스트 상단에서 바로 입력

### 3.2 조직화
- 프로젝트/리스트별 그룹화 (만다라트 프로젝트 트리와 연동, `mandarat.md`)
- 태그, 라벨
- 하위 태스크(sub-task) — Node 트리 구조 그대로 지원
- 필터: 자연어로 필터 조건 서술 → AI가 쿼리로 변환 (예: "이번 주 중요도 높은 것만")

### 3.3 일정/반복
- 마감일(dueAt), 예상 소요시간
- 반복 일정 (매일/매주/매월/커스텀)
- 캘린더 뷰와 통합 표시 (`calendar.md`)

### 3.4 우선순위 & 긴급도
- 긴급도 × 중요도 매트릭스(아이젠하워 매트릭스형 정렬)
- "오늘 할 일" 자동 정렬 뷰

### 3.5 포커스 도구
- 뽀모도로 타이머 (태스크에서 바로 실행)
- 완료 항목 숨기기 / "하지 않음(Won't do)" 처리로 목록 부담 경감

### 3.6 알림
- 항목당 최대 5개, 시간/일/주/월 단위 (`data-model.md`의 `Reminder[]`)

### 3.7 협업 (확장 여지)
- 담당자 지정, 공유 프로젝트 — Phase 6 관리자 모드와 함께 고도화 예정

### 3.8 진행률
- 상위 목표(만다라트 노드) 진행률에 자동 반영 (`data-model.md` §9)

## 4. UI 원칙

- 리스트 뷰가 기본이며, 캘린더 뷰·타임라인 뷰로 전환 가능
- 완료 처리는 체크박스 즉시 반응 (낙관적 UI 업데이트)
- 모바일 하단 탭바의 "투두" 탭에서 진입 (`ui-layout.md`)
