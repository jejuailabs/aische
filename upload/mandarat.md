# mandarat.md — 만다라트 / 프로젝트 트리 모듈

> 원본 기능정의 문서(사용자 첨부)를 기반으로 정리. 데이터 구조는 `data-model.md`의 `Node` 스키마를 그대로 사용한다.

## 1. 핵심 정의

만다라트는 **9칸 목표 관리 도구가 아니라, 목표(Objective)를 계층형 프로젝트 데이터(Node)로 변환하고 이를 투두·캘린더·간트·로그·진행률과 하나의 데이터 모델로 통합 관리하는 시스템**이다.

사용자가 보는 만다라트/트리/간트/캘린더는 각각 다른 기능이 아니라 **동일한 데이터를 다른 관점(View)으로 표현한 것**이다.

## 2. 목표 구조화

- 최종 목표(Root Goal) → 핵심 목표(Core Goals) → 세부 실행 목표(Action Goals)의 부모-자식 계층 구조
- 모든 목표는 독립적인 Node 객체 (그림이 아니라 데이터)

## 3. 무제한 확장

- 기존 3×3(81칸) 만다라트의 한계를 없애고, 모든 목표는 다시 새로운 중심 목표로 확장 가능
- 예: 회사 성장 → 브랜드 구축 → SNS 운영 → 인스타 성장 → 릴스 제작 → 촬영 프로세스 → 촬영 체크리스트
- 트리 전체는 유지되지만 사용자는 현재 집중 중인 목표만 좁혀서 볼 수 있음 (Zoom-in 뷰)

## 4. 노드(Node) 관리

각 노드가 가지는 정보는 `data-model.md`의 `Node` 인터페이스 참조 (ID, 부모/자식, 프로젝트ID, 제목, 설명, 상태, 우선순위, 진행률, 생성/수정/완료일, 담당자, 태그, 예상/실제 소요시간 등).

## 5. 계층 조작

사용자가 트리에서 할 수 있는 조작:

- 접기 / 펼치기
- 이동 (드래그)
- 부모 변경
- 하위 목표 통째로 이동

계층 변경 시 하위 노드 전체가 함께 이동하며 부모-자식 관계가 유지된다.

## 6. 목표 → Task 전환

목표는 텍스트가 아니라 언제든 실행 가능한 Task로 전환된다.
예: "유튜브 성장" → "쇼츠 제작" → "오늘 영상 업로드"

Task는 목표 노드와 동일한 스키마(`Node`, `type: "task"`)로 관리되며, 캘린더/투두 View에도 동시에 노출될 수 있다.

## 7. 상태 및 진행률 자동 집계

- 상태값: 예정 / 진행중 / 대기 / 검토 / 완료 / 보류 / 취소
- 자식 노드 상태가 부모 진행률에 반영 (예: 8개 중 4개 완료 → 부모 50%)
- Task → SubGoal → Goal → Project → Workspace 순으로 진행률 누적 (`data-model.md` §9 참조)

## 8. 일정 연동

모든 노드는 일정 객체(`ScheduleInfo`)를 선택적으로 가지며, 이 정보는 캘린더/간트/오늘 할 일 화면에 동시에 반영된다.

## 9. 다중 View

같은 데이터를 다음 View에서 확인 가능:

- 만다라트 (계층 그리드)
- 트리
- 리스트
- 캘린더
- 간트
- 로그

어느 화면에서 수정해도 전체 View가 즉시 동기화된다.

## 10. 작업 의존성

Task 간 선후 관계 설정 가능 (예: 디자인 완료 → 개발 시작 → 테스트 → 배포). 앞 작업이 끝나야 다음 작업이 활성화되며, 간트 차트에도 동일하게 반영된다. (`data-model.md`의 `dependency` 필드)

## 11. 이력(Log)

생성/수정/삭제/이동/완료/일정변경/담당자변경 등 모든 변경사항을 로그로 기록 (`data-model.md`의 `LogEntry`).

## 12. 우선순위

긴급도/중요도/우선순위 값을 기준으로 "오늘 해야 할 작업"을 자동 정렬.

## 13. 탐색

검색, 태그, 필터, 상태별 보기, 담당자별 보기, 완료 항목 숨기기 기능 제공.

## 14. 시각화

만다라트, Tree, Timeline, Calendar, Progress Bar, Statistics 등 모두 동일 데이터 기반으로 생성.

## 15. AI 확장 인터페이스

모든 노드는 AI와 연결되며, 예를 들어 "카페 창업"이라는 목표 입력 시 AI가 수행 가능한 것:

- 세부 목표 생성
- 실행 계획 생성
- 일정 자동 생성
- 예상 기간 계산
- 위험 요소 분석
- 우선순위 추천
- 회고 작성

AI는 별도 데이터를 관리하지 않고 기존 프로젝트 Node 객체를 기반으로 동작하며, 산출물은 Draft 상태로 생성되어 사용자 Confirm을 거친다 (`core.md` §2-3 원칙).

## 16. 캘린더/투두 항목의 자동 프로젝트 배치

- 캘린더/투두에서 새로 생성된 일정을 AI가 분석해 기존 만다라트 프로젝트에 매칭 시도
- 매칭 성공 시 Draft 형태로 배치 제안 → 사용자 승인(Confirm) 후 반영
- 매칭 실패 시 "미분류함"(`projectId: unsorted`)으로 모아두고, 사용자가 새 프로젝트 생성 또는 기존 프로젝트로 병합
